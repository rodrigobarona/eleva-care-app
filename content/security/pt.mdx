## O Nosso Compromisso com a Segurança

Como plataforma de tecnologia para a saúde, a Eleva.care compreende que a segurança não é apenas uma funcionalidade—é a base da confiança. Tratamos informações sensíveis de saúde de profissionais e pacientes em toda a Europa e América Latina, e levamos esta responsabilidade a sério.

A nossa infraestrutura de segurança é construída sobre fundações conformes com **RGPD**, **LGPD** e **HIPAA**, implementando práticas líderes da indústria para proteger os seus dados em todas as camadas da nossa plataforma.

---

## Certificações e Conformidade

### Padrões de Conformidade Ativos

✅ **RGPD (Regulamento Geral sobre a Proteção de Dados)**  
Conformidade total como entidade baseada em Portugal que serve profissionais de saúde e pacientes da UE.

✅ **LGPD (Lei Geral de Proteção de Dados Pessoais)**  
Conformidade completa com a lei brasileira de proteção de dados (Lei nº 13.709/2018).

✅ **Infraestrutura Preparada para HIPAA**  
Infraestrutura de base de dados conforme com HIPAA através da Neon.tech com Acordo de Associado de Negócios (BAA) executado.

### Certificações de Infraestrutura e Fornecedores

A nossa postura de segurança é reforçada pelas certificações formais dos nossos fornecedores de infraestrutura:

✅ **Infraestrutura Certificada SOC 2 Type II**

- **Clerk** (Autenticação): Certificado SOC 2 Type II
- **Stripe** (Pagamentos): SOC 2 Type II + PCI DSS Level 1
- **Vercel** (Hospedagem): Certificado SOC 2 Type II
- **Neon** (Base de Dados): SOC 2 Type II + conformidade HIPAA

✅ **Práticas de Segurança Alinhadas com ISO 27001**  
A nossa gestão de segurança da informação segue os princípios e melhores práticas da ISO 27001, implementados em todo o nosso ciclo de desenvolvimento, infraestrutura e procedimentos operacionais.

Para informações detalhadas sobre conformidade, visite o nosso [Acordo de Processamento de Dados](/legal/dpa).

## Medidas Técnicas de Segurança

### Encriptação de Dados

**Encriptação em Repouso:**

- Encriptação **AES-256-GCM** para todos os dados relacionados com saúde
- Encriptação ao nível da base de dados através da infraestrutura conforme HIPAA da Neon.tech
- Backups encriptados com recuperação point-in-time

**Encriptação em Trânsito:**

- **TLS 1.3** para toda a transmissão de dados
- Perfect Forward Secrecy (PFS) ativado
- HTTP Strict Transport Security (HSTS) aplicado
- Conexões WebSocket seguras para funcionalidades em tempo real

**Gestão de Chaves:**

- Políticas de rotação segura de chaves
- Integração com Hardware Security Module (HSM) através de fornecedores cloud
- Sem armazenamento em texto simples de chaves de encriptação

### Autenticação e Controlo de Acesso

**Autenticação Multi-Factor (MFA):**

- Verificação baseada em SMS
- Suporte para aplicação de autenticação (TOTP)
- Verificação por email como alternativa
- Obrigatório para contas administrativas

**Gestão de Identidade:**

- Powered by **Clerk** com autenticação de nível empresarial
- Single Sign-On (SSO) com Google
- Opções de autenticação sem senha
- Gestão segura de sessões com timeout automático

**Controlo de Acesso Baseado em Funções (RBAC):**

- Aplicação do princípio do menor privilégio
- Sistema de permissões granulares (Especialista, Cliente, Admin)
- Hierarquias de funções separadas para acesso a dados da plataforma e dados clínicos
- Revisão e auditoria regular de acessos

### Segurança da Infraestrutura

**Residência de Dados na UE:**

- Hospedagem primária da base de dados: **Frankfurt, Alemanha** (Neon.tech)
- Serviços de email: **Irlanda** (Resend)
- Edge computing: **Regiões da UE** (Vercel)
- Cache/Fila: **Regiões da UE** (Upstash)
- Analytics: **Hospedagem na UE** (PostHog)
- Notificações: **Endpoint API da UE** (Novu - eu.api.novu.co)

**Segurança de Rede:**

- Proteção DDoS através da rede edge da Vercel
- Web Application Firewall (WAF)
- Rate limiting em todos os endpoints da API
- Allowlist de IPs para acesso administrativo
- Gateway de API segura com validação de pedidos

**Segurança da Base de Dados:**

- PostgreSQL conforme HIPAA (plano Scale da Neon.tech)
- Registo de auditoria melhorado via extensão **pgAudit**
- Backups automatizados com retenção de 30 dias
- Connection pooling com aplicação de SSL
- Políticas de segurança ao nível de linhas da base de dados

### Segurança de Pagamentos

**Conformidade PCI DSS Level 1:**

- Todo o processamento de pagamentos através do **Stripe**
- Sem armazenamento de dados de cartões de crédito nos nossos servidores
- Tokenização para métodos de pagamento
- Suporte para autenticação 3D Secure
- Deteção e prevenção de fraudes

**Proteção de Dados Financeiros:**

- Chaves de encriptação separadas para dados financeiros
- Trilha de auditoria para todas as transações de pagamento
- Reconciliação e monitorização automatizadas
- Verificação segura de assinatura de webhooks

## Gestão de Vulnerabilidades

### Testes Regulares de Segurança

**Testes de Penetração:**

- Testes de penetração anuais por terceiros
- Avaliações de segurança internas trimestrais
- Scanning automatizado de vulnerabilidades
- Testes de segurança de aplicações web
- Testes de segurança de API

**Remediação de Vulnerabilidades:**

- Vulnerabilidades críticas: 24-48 horas
- Vulnerabilidades altas: 7 dias
- Vulnerabilidades médias: 30 dias
- Vulnerabilidades baixas: 90 dias
- Divulgação pública após remediação

### Divulgação Responsável de Vulnerabilidades

Damos as boas-vindas a investigadores de segurança e hackers éticos para nos ajudar a manter os mais altos padrões de segurança.

**Como Reportar uma Vulnerabilidade:**

1. **Email:** [security@eleva.care](mailto:security@eleva.care)
2. **GitHub Security Advisories:** [github.com/eleva-care/eleva-care-app/security/advisories](https://github.com/eleva-care/eleva-care-app/security/advisories)
3. **PGP Encriptado:** Disponível mediante pedido

**O Que Precisamos:**

- Descrição da vulnerabilidade
- Passos para reproduzir
- Avaliação do impacto potencial
- As suas informações de contacto (opcional)
- Qualquer código proof-of-concept (se aplicável)

**O Nosso Compromisso:**

- Reconhecimento dentro de 48 horas
- Atualizações regulares sobre o progresso da remediação
- Reconhecimento nos nossos agradecimentos de segurança (a menos que prefira anonimato)
- Sem ação legal para investigação de segurança de boa-fé
- Estrita confidencialidade do seu relatório

### Vulnerabilidades em Escopo

✅ Falhas de autenticação e autorização  
✅ Injeção SQL  
✅ Execução de código do lado do servidor  
✅ Escalação de privilégios  
✅ Exposição de dados (PII, PHI, dados de pagamento)  
✅ Sequestro de sessão  
✅ Problemas de segurança de API  
✅ Vulnerabilidades de encriptação  
✅ Falhas de lógica de negócio

### Vulnerabilidades Fora de Escopo

❌ Clickjacking em páginas não sensíveis  
❌ Cross-Site Request Forgery (CSRF) sem impacto  
❌ Ataques que requerem MITM ou acesso físico  
❌ Denial of Service (DoS/DDoS)  
❌ Content spoofing sem modificação de HTML/CSS  
❌ Configuração de email SPF/DKIM/DMARC  
❌ Cabeçalhos de segurança em falta sem exploit  
❌ Melhores práticas SSL/TLS sem vulnerabilidade real  
❌ Rate limiting em endpoints não sensíveis  
❌ Mensagens de erro descritivas sem exposição de dados  
❌ Divulgação de versão de software  
❌ Clickjacking em páginas sem ações sensíveis

**Nota:** Reservamo-nos o direito de designar qualquer vulnerabilidade reportada como fora de escopo com base na avaliação de risco.

### O Que Não Fazer

⚠️ **Não** execute scanners automatizados sem permissão (contacte-nos para um sandbox)  
⚠️ **Não** aceda ou modifique dados de utilizadores além do necessário para demonstrar a vulnerabilidade  
⚠️ **Não** divulgue publicamente a vulnerabilidade antes de termos tido tempo para corrigi-la  
⚠️ **Não** realize engenharia social, phishing ou ataques físicos  
⚠️ **Não** se envolva em qualquer atividade que possa interromper os nossos serviços

## Privacidade e Proteção de Dados

### Minimização de Dados

Recolhemos apenas os dados necessários para fornecer os nossos serviços:

- **Dados de Identidade:** Nome, email, telefone (para autenticação)
- **Dados de Perfil:** Credenciais profissionais, especialidades (para profissionais)
- **Dados Clínicos:** Notas de consultas, gravações de sessões (encriptadas, controladas pelo profissional)
- **Dados de Pagamento:** Métodos de pagamento tokenizados (geridos pelo Stripe)
- **Dados de Utilização:** Analytics da plataforma (anonimizados sempre que possível)

### Retenção de Dados

- **Contas ativas:** Dados retidos enquanto a conta estiver ativa
- **Contas inativas:** Notificação de inatividade de 12 meses, depois eliminação em 24 meses
- **Dados clínicos:** Retidos conforme regulamentos de saúde (varia por jurisdição)
- **Registos de auditoria:** Retenção mínima de 12 meses para investigações de segurança
- **Backups:** Retenção de 30 dias com armazenamento encriptado

### Direitos dos Titulares de Dados

Apoiamos totalmente os direitos do RGPD e LGPD:

- **Direito de Acesso:** Exporte os seus dados em formato portátil
- **Direito de Retificação:** Atualize ou corrija as suas informações
- **Direito ao Apagamento:** Elimine a sua conta e dados
- **Direito de Limitação:** Limite o processamento dos seus dados
- **Direito à Portabilidade:** Transfira dados para outro serviço
- **Direito de Oposição:** Opte por não participar em atividades de processamento
- **Direito à Decisão Automatizada:** Explicação de decisões algorítmicas

**Exercer os Seus Direitos:**  
Email: [privacy@eleva.care](mailto:privacy@eleva.care) ou [dpo@eleva.care](mailto:dpo@eleva.care)

Para mais detalhes, consulte a nossa [Política de Privacidade](/legal/privacy).
