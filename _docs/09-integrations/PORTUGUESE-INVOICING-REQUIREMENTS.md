# Portuguese Invoicing Requirements for Eleva Care

> **Status:** Reference Documentation  
> **Created:** December 16, 2025  
> **Company:** Búzios e Tartarugas, Lda  
> **Country:** Portugal

## Overview

This document outlines the legal invoicing requirements for Portuguese companies operating a B2B services marketplace. As Eleva Care charges platform fees to experts for its services, these fees must be properly invoiced according to Portuguese tax law.

## Legal Framework

### Applicable Legislation

- **Código do IVA (CIVA)** - Portuguese VAT Code
- **Regime do IVA nas Transações Intracomunitárias (RITI)** - EU VAT Transactions
- **Decreto-Lei n.º 28/2019** - Electronic invoicing requirements
- **Portaria 302/2016** - SAF-T requirements

### Key Requirements

1. **Certified Billing Software**: All invoices must be issued through AT-certified software
2. **Sequential Numbering**: Invoices must have unique, sequential numbers
3. **NIF Requirement**: B2B invoices must include buyer's tax ID
4. **Real-time Reporting**: Some transactions require real-time AT communication
5. **SAF-T Export**: Monthly export of all fiscal documents required

## VAT Treatment Matrix

### Services to Experts by Location

| Expert Location | VAT Rate | VAT Treatment   | Legal Basis   | Invoice Text           |
| --------------- | -------- | --------------- | ------------- | ---------------------- |
| **Portugal**    | 23%      | Normal taxation | Art. 18º CIVA | (none required)        |
| **EU B2B**      | 0%       | Reverse charge  | Art. 6º RITI  | "IVA - Autoliquidação" |
| **EU B2C**      | 23%      | PT VAT applies  | Art. 6º CIVA  | (none required)        |
| **Non-EU**      | 0%       | Export exempt   | Art. 6º CIVA  | "IVA - Não sujeito"    |

### Detailed Rules

#### Portuguese Experts (PT)

```
VAT Rate: 23%
Tax Code: NOR (Normal)
Exemption: None
Total = Net + (Net × 0.23)

Example:
- Service fee: €15.00
- VAT (23%): €3.45
- Total: €18.45
```

#### EU Experts (B2B - Intra-Community Services)

```
VAT Rate: 0%
Tax Code: ISE (Isento)
Exemption Reason: M07 - Reverse charge
Exemption Text: "IVA - Autoliquidação (Art. 6º, n.º 6, al. a) do CIVA e Art. 6º do RITI)"

Example:
- Service fee: €15.00
- VAT (0%): €0.00
- Total: €15.00
```

**Important:** The expert's EU VAT number MUST be validated via VIES (VAT Information Exchange System).

#### Non-EU Experts (Export of Services)

```
VAT Rate: 0%
Tax Code: ISE (Isento)
Exemption Reason: M99 - Other exemptions
Exemption Text: "IVA - Não sujeito (Art. 6º, n.º 11 do CIVA)"

Example:
- Service fee: €15.00
- VAT (0%): €0.00
- Total: €15.00
```

## SAF-T Requirements

### What is SAF-T?

SAF-T (Standard Audit File for Tax purposes) is an XML file containing all fiscal data that Portuguese companies must:

1. Export monthly (by the 20th of following month)
2. Submit to Autoridade Tributária
3. Archive for 10+ years

### TOConline SAF-T Generation

TOConline automatically:

- Generates SAF-T files
- Includes all invoices in the export
- Handles submission to AT

### Integration Impact

All invoices created via API are automatically included in SAF-T exports. No additional action required.

## VIES Validation

### What is VIES?

VIES (VAT Information Exchange System) validates EU VAT numbers. Required for:

- Applying reverse charge (0% VAT)
- Intra-community supply reporting

### Validation Process

```typescript
// Before applying reverse charge, validate EU VAT number
const validateEUVat = async (vatNumber: string) => {
  // Use EU VIES service
  const response = await fetch(`https://ec.europa.eu/taxation_customs/vies/checkVatService`, {
    // ... SOAP request
  });

  return response.valid; // true/false
};
```

### Best Practice

1. Validate VAT number during expert registration
2. Re-validate periodically (monthly)
3. Store validation result and date
4. If invalid, apply standard 23% VAT

## Refunds and Credit Notes

### When to Issue Credit Note

Issue `NC` (Nota de Crédito) when:

- Patient requests refund
- Expert refunds appointment
- Platform fee needs reversal

### Credit Note Requirements

- Must reference original invoice number
- Same VAT treatment as original
- Negative amounts

### Example

```
Original Invoice: ELEVA FT 2025/123
Amount: €15.00 + VAT

Credit Note: ELEVA NC 2025/1
Reference: ELEVA FT 2025/123
Amount: -€15.00 - VAT
Reason: "Cancelamento de consulta"
```

## Common Mistakes to Avoid

| Mistake                    | Consequence       | Prevention                     |
| -------------------------- | ----------------- | ------------------------------ |
| Missing NIF on B2B invoice | AT fine           | Always require NIF for experts |
| Wrong VAT for EU experts   | Double taxation   | Validate VIES before 0% rate   |
| Sequential gap in numbers  | AT audit flag     | Use certified software         |
| Late SAF-T submission      | €375-€22,500 fine | Automate monthly export        |
| Missing ATCUD              | Invalid invoice   | Use certified software         |

## Expert Data Collection Requirements

### Minimum Required Fields

```typescript
interface ExpertFiscalData {
  // Required for all experts
  fiscalNif: string; // Tax ID
  fiscalBusinessName: string; // Legal name
  fiscalCountry: string; // ISO country code

  // Required for invoicing
  fiscalAddress: string;
  fiscalPostcode: string;
  fiscalCity: string;

  // For EU experts
  euVatNumber?: string; // Full EU VAT (e.g., ES12345678B)
  euVatValid?: boolean; // VIES validation result
  euVatValidatedAt?: Date; // Last validation date

  // Business type
  isCompany: boolean; // true = B2B, false = B2C
}
```

Add to expert onboarding flow:

1. Country selection
2. NIF/VAT number input
3. Business name
4. Full address
5. Automatic VIES validation for EU
